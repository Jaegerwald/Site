Bug report guide

This guide is intended for reporting bugs within my published software. Please follow it closely. This guide is not for software where the source is available on GitHub. Please open an issue on GitHub in that case instead.

1 — capture screenshots/walkthroughs to reproduce the bug

Optionally also describing the issue in words alongside the images is also good.

2 — give log files or similar

If this is an option, attach all files that could help.

3 — write the e-mail

Send a mail to jae@sharkb.yt formatted like this:

Subject: BUG REPORT | [short description of your issue in ~3-7 words]
Software: -----
Version: -----

Long description of your issue

[Images at the bottom]
Attached: all important files for your issue
